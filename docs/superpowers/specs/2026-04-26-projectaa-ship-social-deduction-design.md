# ProjectAA — 선박 배경 1인칭 소셜 디덕션 게임 설계안

**날짜:** 2026-04-26  
**엔진:** Unreal Engine 5.5  
**목표:** 포트폴리오용 버티컬 슬라이스 (핵심 루프 1판 플레이 가능)

---

## 개요

선박을 배경으로 한 1인칭 소셜 디덕션 게임. 승무원은 태스크 수행 및 단서 탐색으로 스파이를 찾아내고, 스파이는 방해 및 제거로 승무원을 줄여 나간다. 툰쉐이더 비주얼, 모듈러 스켈레탈 메시 커스터마이징, 온라인 멀티플레이어(2~4인) 지원.

---

## 섹션 1: 전체 아키텍처

### 시스템 구성

| 클래스 | 역할 |
|--------|------|
| `BP_GameMode` | 서버 전용: 역할 배정, 게임 상태 관리 |
| `BP_GameState` | 전체 동기화: 페이즈, 태스크 현황 |
| `BP_PlayerState` | 개인 정보: 역할(승무원/스파이), 의상, 생존 여부 |
| `BP_Character` | 1인칭 캐릭터 (기존 셋팅 기반 확장) |
| `BP_TaskActor` | 상호작용 가능한 태스크 오브젝트 |
| `BP_ClueActor` | 스파이 행동 후 생성되는 단서 |
| `UI_HUD` | 태스크 목록, 미니맵, 투표 화면 |

### 게임 페이즈 흐름

```
로비 (의상 선택)
  → 준비 (역할 비밀 배정)
  → [탐색/태스크 페이즈] ↔ [긴급회의/투표 페이즈]
  → 승리 조건 달성
  → 결과 화면
  → 로비
```

### 승리 조건

- **승무원 승리:** 모든 태스크 완료 OR 스파이 투표 추방
- **스파이 승리:** 생존 승무원 수 ≤ 스파이 수 (과반 달성)

## 섹션 2: 맵 & World Partition 구조

### 선박 3구역 레이아웃

```
[갑판 / Deck]          ← 외부, 바다 뷰, 탈출 포드 위치
      ↕ 계단/통로 (심리스)
[선내 / Interior]      ← 메인 홀, 회의실, 객실 복도
      ↕ 계단/통로 (심리스)
[기관실 / Engine Room] ← 좁고 어두움, 기계장치 태스크 집중
```

- 각 구역은 **World Partition 스트리밍 셀**로 분리
- 플레이어 인접 시 자동 로드/언로드, 로딩 화면 없는 심리스 이동
- 바다 배경: UE5 **Water Plugin** 활용 (갑판에서만 노출)

### 툰쉐이더 적용

- 캐릭터: 커스텀 **Post Process Material** (아웃라인 + 셀쉐이딩)
- 아웃라인 색상·굵기: 파라미터화 (역할/팀별 구분 가능)
- 환경: 스타일라이즈드 라이팅으로 보완, 캐릭터에 집중 적용

### 태스크 배치 (구역별)

| 구역 | 태스크 예시 |
|------|------------|
| 갑판 | 신호등 수리, 밧줄 고정 |
| 선내 | 버튼 조작, 보고서 제출 |
| 기관실 | 드릴 작업, 해머 수리 |

### 단서 시스템

- 스파이가 승무원 제거 시 해당 위치에 `BP_ClueActor` 스폰
- 승무원이 단서 조사 시 마지막 목격 구역 정보 획득
- 단서는 일정 시간 후 자동 소멸 (스파이에게 시간 여유 부여)

## 섹션 3: 역할 & 스파이 능력

### 역할 시각화

| 상태 | 본인 화면 | 타인 화면 |
|------|----------|----------|
| 승무원 | 파란 HUD 테두리 | 일반 캐릭터 |
| 스파이 | 빨간 HUD 테두리 | 일반 캐릭터 (위장) |
| 사망 | 회색 유령 시점 | 시체만 보임 |

### 스파이 능력

- **방해:** 태스크 오브젝트 일시 비활성화 (서버에서 처리)
- **제거:** 근접 시 E 키 → 애니메이션 재생 후 승무원 사망
- **환기구 이동:** 특정 위치에서 갑판 ↔ 기관실 빠른 이동

## 섹션 4: 네트워크 & 투표 시스템

### 네트워크 구조 (Dedicated Server + OSS 추상화)

```
Dedicated Server (별도 빌드)
├── GameMode: 역할 배정, 승리 조건 판정
├── GameState: 페이즈, 태스크 현황 (전체 복제)
└── PlayerState: 역할, 생존 여부 (개인별 복제)

Client × N
└── PlayerController → Server RPC → DS 검증 후 처리
```

**OSS 레이어 전략:**

| 단계 | Subsystem | 용도 |
|------|-----------|------|
| 개발/테스트 | `NULL` | 로컬 IP로 DS 접속 |
| 완성/배포 | `Steam` | Steam 매칭, 로비 |

- `OnlineSubsystem` 인터페이스만 사용 → `DefaultEngine.ini` 수정만으로 스왑
- Session 생성/검색/입장 모두 OSS API로 통일

### 주요 RPC 패턴

- 태스크 완료: `Server_CompleteTask()` → `GameState` 업데이트 → 전체 브로드캐스트
- 제거: `Server_EliminatePlayer()` → 서버 검증 후 처리
- 단서 생성: 서버에서 `BP_ClueActor` 스폰 → 자동 복제

### 긴급회의 & 투표 시스템

```
트리거 (회의 버튼 or 시체 발견)
  → Server: 탐색 페이즈 일시 정지
  → GameState: 투표 페이즈 전환 (타이머 60초)
  → 모든 클라이언트: 투표 UI 표시
  → 각 플레이어 Server_CastVote(TargetID) 호출
  → 60초 후 서버 집계 → 최다 득표자 추방 or 무효
  → 탐색 페이즈 재개
```
